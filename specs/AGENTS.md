# Padrões para Criação de Specs

Este diretório contém as especificações técnicas das features antes de sua implementação. 

## Estrutura Padrão de uma Spec (.md)

Ao criar uma nova especificação, utilize o seguinte formato:

1. **Objetivo**: Descrição curta e direta do que será construído e seu propósito.
2. **Escopo**: O que faz parte da feature (e o que está explicitamente fora).
3. **Decisões Técnicas**: Bibliotecas, ferramentas, modelos de dados e padrões a serem utilizados.
4. **Passo a Passo (Plano de Ação)**: Lista de tarefas granulares e ordenadas para guiar a implementação.

## Regras de Execução (Para Agentes)

- **Sempre** leia a spec inteira antes de iniciar a implementação ou criar arquivos.
- Siga o plano de ação estritamente na ordem definida.
- Valide as dependências e o contexto atual do projeto antes de seguir as decisões técnicas.
- Em caso de bloqueios ou ambiguidades na spec, pare e peça esclarecimentos ao usuário.
